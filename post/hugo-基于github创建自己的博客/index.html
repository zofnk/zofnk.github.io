博客搭建可行性方案

就目前个人了解到的搭建静态博客框架方案大致有如下几种：

  • Hexo
  • Hugo
  • Jekyll
  • Ghost

略微统计了下其各自在Github中的表现：

Github stars issues(open) issues(close) language multilingual free
Hexo 28.6k 194 3083 JavaScript √ √
Hugo 39.3k 379 3665 Go - √
Jekyll 39.1k 85 3954 Ruby - √
Ghost 31.7k 51 5517 JavaScript - ×

对比各自的表现来看：

  • Jekyll相对其他三个更活跃，而且免费。
  • Ghost收费，在有其他3个框架相对比下就显得不建议使用。

前三个我都有搭建起来过,鉴于我本身不是前端开发,在没有多少前端基础的情况下,三个框架都相对比较容易上手,搭建到部署也简易快捷,在这先记录下用Hugo的搭建过程。

Hugo是由Go语言实现的静态网站生成器。简单、易用、高效、易扩展、快速部署。

1. 下载二进制文件

  • 到Hugo Github下的 releases下载自己系统对应的二进制文件。我的是WIN10所以选择hugo_0.59.1_Windows-64bit.zip。

2. 修改环境变量PATH

  • 解压得到hugo.exe,并将它添加进系统环境变量当中。

3. 验证安装

  • Hugo,win + R后输入cmd后打开命令行界面,输入hugo version后,终端输出Hugo Static Site Generator v0.59.1-D5DAB232 windows/amd64 BuildDate:时间即可。

4. 创建站点

  • 继续在命令行下进入自己创建的站点文件夹
  • 输入hugo new site myBlog后回车,可以看到一个叫myBlog的目录后,输入cd myBlog,进入目录。

5. 安装主题

  • 在Hugo Themes或者Github中找一个自己满意的样式,以下以我自己选择的Even举例说明。

  • 直接下载even-master.zip 文件,或者在命令行中输入。

    1

    $ git clone https://github.com/olOwOlo/hugo-theme-even themes/hugo-theme-even

  • 下载完毕后在myBlog文件夹下打开config.toml,将theme的值修改为刚下载完的主题文件夹名称后保存退出。

    1

    theme = "hugo-theme-even"

因为Even有自带的配置文件,所以可以将myBlog/themes/hugo-theme-even/exampleSite/中的config.toml直接覆盖至myBlog/目录下。

6. 发表文章

  • 在命令行中输入hugo new posts/Hello_World.md后回车(这里的post要根据主题的设定而定,Even需要将posts更改成post)。

  • 在myBlog/content/posts/下会生成Hello_World.md的文件,这就是需要撰写的第一篇文章了,并且需要了解Markdown的相关语法知识。

  • 打开Hello_World.md后可以看到已经有一段自带的头部信息了。

    1
2
3

    title:  #标题
date:  #时间
draft:  #草稿

  • 在编写完自己要写的内容后把draft的值改成false

  • Even Theme中有自带的文章,可以用来预览一番,拷贝myBlog/themes/hugo-theme-even/exampleSite/content/目录下的所有md文件,黏贴至myBlog/content/posts/目录下即可。

8.部署Github Pages

  • 登录你的Github,并创建一个空的Repository,在Repository name框中输入${yourusername}.github.io,点击Creat repository完成创建。

  • 打开站点根目录,打开config.toml文件,将其中baseURL的值修改成刚刚创建的git repository name。

    1

    baseURL = "${your_repository_url}"

  • win+R打开命令行并进入站点文件夹(cd ./myBlog),执行hugo命令。执行完毕后会在站点目录下看到public文件夹,这个就是我们需要push到git的东东了。

  • 用cd public进入public文件夹,用以下git命令来对文件夹进行push操作

    1
2
3
4

    $ git remote add origin ${your_repository_url}
$ git add -A
$ git commit -m "${log}"
$ git push -u origin master

  • 进入刚刚创建的git项目主页,点击home tab最后一项setting,进入设置页面,在options模块下找到GitHub Pages,可以看到提示Your site is read to be published at https://${your_url},可以点击url看看是否已经创建成功,如果创建失败,则看看是否和站点配置文件的baseURL一致,如果不一致的话,需要修改站点的baseURL,再执行：

    1
2
3

    $ hugo
$ git commit -m "fix baseURL"
$ git push

然后再打开你的博客地址,看看是否已经创建完成。

9.域名解析

  • 如果觉得${name.github.io}有点不够意思,可以在阿里云或者腾讯云里购买自己想要的域名。

  • 到控制台中找到要解析的域名,点击解析

  • 创建2条记录

记录类型 主机记录 解析路线 记录值 TTL
A CNAME www 默认 ${your_repository_url} 默认
B CNAME @ 默认 ${your_repository_url} 默认

  • 回到gitRepository -> setting -> GitHub Pages , 在Custom domain中填入你的域名,点击save即可 , 就能在项目根目录下看到CNAME文件,记录值就是你刚输入的域名了。

  • 过几分钟可以打开你的域名看看是否已经解析完成。

10.再来搞点事!打开域名后发现左边有个!咋办?

这是域名没有加载ssl证书,即不是https的url。 如何让自己的域名加个锁呢？ 证书有很多途径获取,免费的收费的都有,在这还是以阿里云的免费证书为例

  • 打开阿里云的证书选购页面。

  • 在选择证书类型当中选择免费型DV SSL,点击购买,因为是0元所以可以直接一路往下点。

  • 支付完成之后来到证书控制平台页面,找到刚刚购买的证书,此时证书还是处于未签发状态。

  • 点击右侧的证书升级,并在证书绑定域名处填写刚刚购买的域名,直接点击下一步 -> 验证 -> 提交审核,完成证书的提交审核流程,接下来就是等待证书的审核通过,一般都是1~2天就会完成,耐心等待吧。

  • 在审核通过后,清除一下浏览器的缓存,ctrl + R刷新浏览器再看看链接是否变成了https,左边的图标也变成了一个锁,到此就大功告成!
